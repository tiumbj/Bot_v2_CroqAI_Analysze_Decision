Plain-language state summary:
1. System starts at Bootstrap/Init and enters the main entrypoint.
2. main() selects one of three runtime modes:
   - RUN_POSITION_MONITOR=1
   - RUN_PRODUCTION_RUNTIME=1
   - RUN_CANDIDATE_TO_GROQ_SMOKE=1
3. In production runtime, the loop is:
   Load Market Data -> Build Snapshot -> Detect Raw Candidates -> Finalize Candidates -> Dedup Check -> Groq Prompt -> Groq Request -> Parse AI Response -> Write ai_decisions.jsonl -> next cycle.
4. In position monitor mode, the loop is:
   Position Exit Eval -> Close Execution Gate -> Write position_monitor.jsonl -> next cycle.
5. Confirmed production gap from the uploaded file:
   processed_setups is in-memory only, so restart can re-send the same setup to Groq.
6. Next strict production-phase step from the uploaded file:
   add persistent setup dedup / restart-safe runtime state in app/main.py.
